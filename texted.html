Text editor features

Languages Supported
Most of the text editor features in Viasfora are available for the following languages:
  • C/C++
  • C#
  • VB
  • F#
  • JavaScript
  • SQL
  • TypeScript
  • Python
  • PowerShell
Rainbow Parentheses
New in v1.1 Inspired by the Vim plugin of the same name, this feature makes it easy to keep track of nested braces. You can customize the colors used for the 4 nesting levels.
Caret Position Highlight
Viasfora can highlight the current line in the text editor, like the built-in one in VS2013, but also available in previous Visual Studio versions. Customize the look through the "Viasfora Current Line" entry in the Fonts and Colors options page.

New in v1.5 You can also highlight the current column in the text editor:

Both of these features come disabled by default, and can be enabled and configured independently of one another.

Control Flow Classifier
Customize how control-flow related keywords are rendered in the VS editor. You can choose which keywords are selected in each language and customize the presentation through the "Viasfora Flow Control Keyword" entry in the Fonts and Colors options page.
LINQ Classifier
Customize how Language Integrated Query (LINQ) keywords are rendered in the VS editor. You can choose which keywords are selected for each language and customize the presentation through the "Viasfora Query Operator" entry in the Fonts and Colors options page.
Visibility Classifier
Customize how visibility related keywords in are rendered in the VS editor. You can choose which keywords are selected in each language and customize the presentation through the "Viasfora Visibility Keyword" entry in the Fonts and Colors options page.
String Escape Characters
Easily recognize escape sequences in your string literals in C-based languages. Customize the presentation through the "Viasfora String Escape Sequence" entry in the Fonts and Colors options page.
Presentation Mode
New in v1.3 When Presentation Mode is turned on, all text views (including the Output Window) will default to a predefined zoom level (150% by default), without having to change it manually.

It also allows you to define a zoom level different from 100% for when Presentation Mode is turned off.

To turn Presentation Mode on/off, use the View -> Presentation Mode option.

Modelines
New in v1.4 Support for basic Vim-style modelines in the supported file formats:

// vim: et:sw=8:ts=8

Find more details here.

User-Defined Outlining
New in v1.5 You can now define arbitrary outlining (collapsible) regions on text files. To add a new region, just highlight the block of text, then select Edit -> Viasfora -> Add Outlining

To later remove it, just put the caret inside the outlining region and use the Remove Outlining command instead. If the caret is inside of multiple regions, the one with the start position closest to the caret will be removed.

User-defined outlining regions are persisted across sessions through the solution *.suo file.
